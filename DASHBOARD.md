# דשבורד FinOps

מסמך זה מתאר את הממשק החדש, הארכיטקטורה והחלטות ה־UX של הדשבורד המקומי.

## ארכיטקטורה בטוחה

שכבת הנתונים וה־API נשארו ללא שינוי ב־`src/dashboard/server.ts` וב־`src/dashboard/data.ts`. הממשק החדש נמצא בתת־פרויקט מבודד ב־`src/dashboard/web` ובנוי עם React, Vite, Tailwind, shadcn/ui (Radix) ו־Recharts.

השרת מגיש את `src/dashboard/web/dist` אם קיים build תקין. הממשק המקורי נשמר ב־`src/dashboard/index.html` כ־fallback וניתן להפעילו עם `npm run dashboard:legacy`. כך כשל או build חסר אינם פוגעים בגישה לנתונים.

כל התלויות מוצמדות לגרסאות מדויקות ונשמרות ב־`src/dashboard/web/package-lock.json`. לא נטענים scripts, fonts או נכסים מרשת בזמן השימוש. הדשבורד ממשיך להיקשר ל־`127.0.0.1` בלבד.

## היררכיית המידע

בדסקטופ הניווט יושב בסרגל צד ימני קבוע, בהתאם לכיווניות העברית; בטאבלט הוא עובר לשורת טאבים ובמובייל ל־Sheet. הניווט מחולק לחמישה מרחבי עבודה:

1. **תמונה פיננסית** — נטו חודשי, מגמת הכנסות/הוצאות, תמונת מצב מאוחדת, חריגות, קטגוריות מובילות, שינויים והמלצות.
2. **חשבונות וכרטיסים** — הפרדה בין כרטיסים לעו״ש, המחשת כלל מניעת הכפל, כרטיסים חזותיים ופירוט קטגוריות ומוקדי הוצאה לכל מקור.
3. **הוצאות** — תמהיל קטגוריות, תור סיווג, טבלת קטגוריות ותנועות עם חיפוש, סינון, paging ופעולות לימוד ליועץ.
4. **יעדים והתחייבויות** — יעדים, מנויים, הוצאות קבועות ותשלומים עתידיים באותו ציר זמן.
5. **ניהול** — שאלות פתוחות, הוראות שמורות, דוחות מקומיים ובריאות האוטומציה.

הסדר בכל מסך הוא: תמונת מצב → חריגה או החלטה → פירוט → פעולה. המסך הראשי אינו מנסה להציג את כל הנתונים, אלא מפנה ל־drill-down עם הפילטר הרלוונטי.

## שפת עיצוב

זהות "מכשור מדויק": גרפיט קריר נושא את השלדה, ואמבר הוא **אות** יחיד וחם — פעולות, בחירה, פרוגרס והדופק החי. שני המצבים חולקים את אותו היגיון כדי שמעבר ביניהם ייקרא כמוצר אחד תחת שתי תאורות:

- **מצב בהיר:** פלטינה קרירה (`oklch` בגוון ~250, כמעט בלי כרומה), כרטיסים לבנים עם מסגרות hairline, ורמז אמבר עדין בפינת העמוד.
- **מצב כהה (מראה הדגל):** גרפיט־דיו — הכרומה נשארת ≤0.015 כדי שהקנבס ייקרא כדיו ולא כ־navy מותגי. זוהר אמבר אחד בפינה העליונה ("האור בחדר") מול נגד־זוהר פלדה קריר בצד השני — זה מה ששומר על הקנבס קריר במקום לגלוש חזרה לאספרסו. המשטחים שקופים־למחצה (88% אטימות) **בלי** `backdrop-filter` פר־כרטיס — תריסר משטחים עם blur חי גררו את כל פריים הגלילה; רק הכותרת הדביקה מטשטשת.
- **ארבעת ה־KPI המאוחדים (`InsightRow`):** תאי מכשור שטוחים (`.metric-cell`) — שקע מוצלל בתוך המשטח, בלי מסגרת ובלי גרדיאנט. הערך הוא הדבר הבולט בתא; סטטוס מוצג כנקודה+מילה, לעולם לא בצבע בלבד. (אריחי הגרדיאנט הקודמים נקראו כתבנית SaaS — בדיוק האסוציאציה שהמוצר מנסה לא לשדר. עדיין בדיוק ארבעה, לא שמונה כרטיסים.)
- **טיפוגרפיה:** ‏`Geist Variable` לספרות וללטינית, ‏`Heebo Variable` לעברית — הדפדפן נופל גליף־גליף, כך שכל מספר במסך מקבל את הגרוטסק ההנדסי ואת אותו רוחב ספרה. שתי המשפחות מוצמדות ונטענות מקומית (fontsource), בלי רשת. `.numeric` מוסיף `tabular-nums`‏, LTR מבודד ו־tracking ‏-0.01em.
- אין eyebrow מעל כותרות — הכותרת והתיאור נושאים את המסך (תבנית ה"תווית העוקבת" הוסרה במכוון).
- ירוק ואדום משמשים למשמעות פיננסית בלבד, על משטחים ניטרליים; המשמעות מגובה גם בטקסט ובאייקון. הם מוגדרים כטוקנים (`--positive`/`--negative`, וכ־`text-positive`/`text-negative` ב־Tailwind) ולא כ־emerald/rose של Tailwind — אלה גוונים קרים שיצאו מהמשפחה. נקודות סטטוס שאינן רווח/הפסד (טוסט הצלחה, "מחובר", בריאות מערכת) ממשיכות להשתמש בירוק משלהן ואינן חלק מהסמנטיקה הזו.
- **תנועה = מצב בלבד, 150–250ms:** נקודת ה־live נושמת כי הנתונים באמת מתרעננים כל 5 שניות; מעבר מסך מקבל fade של 220ms‏ (`key={tab}`); כפתור הרענון מסתובב בזמן טעינה. הכל קורס לרגעי תחת `prefers-reduced-motion`.
- **כרטיס האשראי הפיזי** (`.card-visual`): גרפיט־מתכת עם גליץ אמבר מהקצה וצ'יפ זהב (`.card-chip`) — תמיד כהה, בשני המצבים.
- רדיוס הבסיס `0.75rem` (כרטיסים ~17px). צפיפות גבוהה נשמרת בטבלאות, לא בכרטיסי ההחלטה.
- ניגודיות אומתה חישובית על צמדי הטוקנים בשני המצבים: כל טקסט גוף ≥4.5:1 (הטווח בפועל 5.2–18.6 בבהיר, 5.6–16.7 בכהה). `--muted-foreground` הכהה הורם ל־L ‏0.72 כי על תאי המכשור המוארים ב־3.5% לבן הוא נחת על קו ה־AA.
- כל רכיבי הטופס, התפריטים, הדיאלוגים והאישורים מבוססים על primitives נגישים של shadcn/ui.
- קיימים מצב כהה, focus-visible ו־`prefers-reduced-motion`; ‏`theme-color` כפול ב־HTML לפי ערכת הצבעים של המערכת.

כל הצבעים מוגדרים כמשתני CSS ב־`index.css` פר־theme. אין צבעים קשיחים ב־`App.tsx` — גם צירים, רשתות ו־tooltips קוראים מהטוקנים, אחרת הם נשברים במצב כהה.

## גרפים

- **תזרים לאורך זמן:** תרשים משולב — עמודות להכנסות ולהוצאות וקו לנטו. מקרא גלוי מופיע מעל הגרף, ציר הסכומים מוצג, וטקסט קצר מסביר כיצד לקרוא אותו.
- **קטגוריות:** bar chart אופקי ומדורג בשני המסכים. לכל שורה label ישיר, סכום, אחוז ופס יחסי; לכן אין צורך לנחש התאמה בין פלח למקרא. לחיצה מסננת את התנועות.
- **עומס לאורך החודש:** area chart לפי שבוע בחודש, כדי לזהות האם רוב הכסף יוצא בתחילתו או מתפזר.
- **יעדים ותשלומים:** progress bars עדיפים על גרף נוסף, משום שהשאלה היא התקדמות מול יעד ולא מגמה.

פלטת הנתונים הקטגורית (`--data-1..8`) היא **רמפ חם יחיד** — זהב, ענבר, כתום, ברונזה — ולא קשת גוונים. הקטגוריות תמיד מתויגות וממוינות לפי גודל, ולכן הן צריכות להיות **ניתנות להבחנה** ולא בלתי־נשכחות כל אחת לחוד; בסקירה מוצגות שש בכל רגע נתון. ההפרדה נשענת בעיקר על **בהירות** ולא על גוון, כך שהסדר שורד גם בגווני אפור וגם בעיוורון צבעים — כל שני שלבים סמוכים נבדלים בלומיננס. אותה קטגוריה מקבלת אותו צבע בסקירה, בפילוח ובטבלה.

סדרות התזרים: **הכנסות בירוק, הוצאות באדום** — אותה משמעות פיננסית שנושאים כל שאר מספרי הרווח/הפסד בעמוד, כדי שעמודה ומספר לא יסתרו זה את זה. שני הגוונים מוטים לכיוון החם (ירוק ב־hue 152 ולא 165, אדום ב־30 ולא 15) כך שהם יושבים באותו חדר עם הרמפ הענברי בלי לאבד את המשמעות. הנטו נשאר קו ענברי.

**טקסט ומילוי הם טוקנים נפרדים** (`--positive`/`--negative` מול `--series-income`/`--series-expenses`), כי יש להם עבודות שונות: לטקסט צריך ניגודיות מול **הרקע**, ולעמודות צריך ניגודיות **זו מול זו**. אדום־ירוק הוא עיוורון הצבעים הנפוץ ביותר, ולכן העמודות מופרדות ב**בהירות** (יחס ~2.2:1 ביניהן) ולא בגוון בלבד — כך הן קריאות גם בדאוטרנופיה וגם בגווני אפור. שימוש בטוקני הטקסט לעמודות היה מציב ירוק ואדום כמעט באותו לומיננס. העמודות והשטח ממולאים בגרדיאנט SVG — משתני CSS בגרדיאנט חייבים לעבור דרך `style={{ stopColor }}` ולא כ־presentation attribute. צבעי סטטוס נשארים נפרדים: ירוק לחיובי, אדום לשלילי וכתום לאזהרה.

**אנימציית Recharts מכובה בעמודות ובשטח (`isAnimationActive={false}`)** — עם ה־polling כל 5 שניות האנימציה מופעלת מחדש בכל רענון ומשאירה את העמודות במצב ריק, כלומר גרף שלא מצייר כלום. אין להחזיר אותה בלי לפתור את זה.

ההחלטות מבוססות על הנחיות Carbon למקרא ופלטות קטגוריות ועל המלצות Power BI להעדיף bars על charts מעגליים בהשוואת ערכים.

## התנהגות שנשמרה

- polling כל חמש שניות ועדכון ידני לפי דרישה. **כשל ברענון רקע אינו מוחק את המסך:** אם כבר יש נתונים טובים, הם נשארים מוצגים ומופיע באנר "הנתונים אינם מתעדכנים כרגע" עם כפתור ניסיון חוזר, שנעלם מעצמו כשהחיבור חוזר. רק כשל בטעינה הראשונה (כשאין על מה ליפול) מציג שגיאה חוסמת. כשל רשת מתורגם להודעה בעברית ולא מדליף את המחרוזת `Failed to fetch` של הדפדפן.
- בחירת חודש, חיפוש, סינון, paging ו־drill-down.
- accept/dismiss/done להמלצות והסתרת התראות.
- סיווג ספק או שירות, סימון העברה פנימית, הערות ושאלות ליועץ.
- יצירה, עדכון, השהיה, השלמה וארכוב של יעדים.
- יצירת דוחות שבועיים וחודשיים מקומיים.
- אין פעולת תשלום או העברת כסף.

## כיסוי מידע מול הממשק הקודם

השדרוג החזותי אינו מחליף נתונים בתצוגות מקוצרות בלבד. לאחר בדיקת parity מול `src/dashboard/index.html`, הממשק החדש מציג גם את שכבות העומק הבאות:

- **סקירה:** יתרת עו״ש, כרית נשימה, קצב הוצאה, ממוצע הוצאות ונטו מצטבר לשישה חודשים מאוחדים בתוך תמונת המצב; הנטו החודשי מוצג בקו של גרף התזרים; סקירת היועץ וכל פרטי ההמלצות נשמרו.

**סקירת היועץ היא הקשר בלבד, לא משימות.** `urgent_actions` הוצגו בעבר כתיבות "פעולה" ענברות, אבל הן מחרוזות טקסט בלי `id` ובלי `status` — אין מה לאשר, לדחות או לנווט אליו, ולכן פריט שכבר טופל המשיך להופיע כדחוף בלי דרך לנקות אותו. כל דבר שדורש פעולה חייב להופיע כ**המלצה**, שיש לה מחזור חיים מלא (`new/accepted/dismissed/done`). השדה נשמר בסכמה לתאימות לאחור עם סקירות שכבר נשמרו ב־`agent_memory`, אך אינו מרונדר.
- **חשבונות:** הכנסות, העברות, עמלות, מגמה מול הממוצע, חמש קטגוריות וחמישה מוקדי הוצאה לכל חשבון, ופירוט מלא של ההעברות שהוחרגו כדי למנוע כפל חיוב.
- **הוצאות:** טבלת קטגוריות מלאה עם כמות חיובים, מגמה, סכומי שישה חודשים ושני אחוזי השתתפות; טבלת ספקים ושירותים מלאה עם חיפוש, סינון, כמות חיובים, חודשים פעילים ופעולות; סינון תנועות גם לפי חשבון.
- **התחייבויות:** כל החיובים הפעילים והרדומים, ממוצע מול חיוב אחרון, חריגת מחיר, כל תוכניות התשלומים וסכום העסקה הכולל.
- **ניהול:** שאלות פתוחות וסגורות עם תשובות, מצב חיוב אחרון של הוראות ביטול, ופירוט הרצות ומשיכות אחרונות.

טבלאות ורשימות ארוכות נשארות מלאות, אך מוצגות בתוך אזור גלילה תחום עם כותרת קבועה. כך המידע אינו נעלם והמסך אינו מתארך ללא גבול.

## החלטות קונסולידציה

לכל שאלה פיננסית יש תצוגת מקור אמת אחת. תצוגות נוספות הן קיצור דרך בלבד ולא עותק נוסף של אותו מידע:

- גרף הנטו הנפרד הוסר משום שקו הנטו כבר מופיע בגרף ההכנסות וההוצאות.
- כרטיס מוקדי ההוצאה הנפרד הוסר. טבלת "מוקדי ההוצאה של משק הבית" היא המקור היחיד וניתן למיין אותה לפי החודש, שישה חודשים, מגמה או שם. השפה מכוונת לניהול משק הבית; "ספק / שירות" מופיע רק ברמת הפירוט שבה הוא מתאר את מקור החיוב בפועל.
- גרף הקטגוריות נשאר כתמונת מצב; הטבלה ההיסטורית המלאה נמצאת תחת “כל הקטגוריות וההשוואה ההיסטורית” ונפתחת לפי צורך.
- מרכז החריגות מציג התראות בלבד. המלצות היועץ מופיעות פעם אחת, עם כל הביסוס והפעולות.
- נתוני היתרה, החיסכון, התחזית, הקצב, כרית הנשימה והממוצע מוזגו לארבע שורות הקשר במקום שמונה כרטיסי KPI.
- פירוט העברות שהוחרגו מוצג תחת disclosure סגור, מפני שהסכום הכולל כבר מופיע במדדי החשבונות.
- סיכום המנויים והחיובים הקבועים מופיע בכותרת הרשימה עצמה; בכל שורה הממוצע החודשי הוא הנתון הראשי והחיוב האחרון הוא ההקשר.

## Responsive ונגישות

בדסקטופ הממשק מנצל רוחב לתרשימים, טבלאות והשוואות. במובייל הניווט עובר ל־Sheet, המדדים מוצגים בשני טורים, האזורים הרחבים נערמים והטבלאות נגללות בתוך המשטח בלי להרחיב את המסמך.

רכיבי הדיאלוג כוללים focus management, תיאורי שדות ואישור מפורש לפעולות מחיקה. אייקונים דקורטיביים אינם מחליפים תווית, וכפתורי icon-only מקבלים `aria-label`.

## פקודות

```bash
npm run dashboard         # build של React + הפעלת השרת המקומי
npm run dashboard:build   # build של ה־frontend בלבד
npm run dashboard:legacy  # הפעלת ממשק ה־fallback
```

## אימות

```bash
npm run dashboard:build
npm run build
npm run brief:selftest
npm run goals:selftest
FINOPS_AUTOMATION_ENABLED=1 npm run automation:selftest
npm run ingest:selftest   # מושך מהבנק בפועל — לא בדיקה יבשה
node scripts/dashboard-visual-check.mjs
```

שתי הערות שחוסכות דקירה בחושך: ‏`automation:selftest` **יוצא בשקט בלי הדגל** `FINOPS_AUTOMATION_ENABLED=1` (האוטומציה כבויה כברירת מחדל), ו־`ingest:selftest` הוא היחיד שנוגע ברשת — הוא ייכשל כשספק חסום, וזה לא רגרסיה בדשבורד. `dashboard-visual-check` דורש שרת פעיל; אפשר להצביע אותו לפורט אחר עם `FINOPS_DASH_URL`.

בדיקת הדפדפן מצלמת את כל חמשת המסכים בדסקטופ ואת הניווט במובייל, מאמתת שאין overflow אופקי ואוספת שגיאות console. הצילומים נשמרים ב־`/tmp/finops-dashboard-verify` ואינם נכנסים ל־git.
